1. Понятия Тестирование (в узком и широком смысле)

Определение: Процесс оценки системы для обнаружения различий между фактическим и ожидаемым поведением.

Узкий смысл:

  • Выполнение тест-кейсов для проверки функциональности
  • Пример: Валидация формы регистрации

Широкий смысл:

  • Комплексная оценка всех характеристик качества
  • Пример: Аудит безопасности веб-приложения

2. Понятия отладка (debugging)

Определение: Технический процесс идентификации, анализа и устранения причин ошибок в программном коде.

  1. Этапы:
    • Воспроизведение ошибки
    • Анализ стека вызовов
  2. Инструменты: Debugger, логирование

3. Что является целью тестирование и что оно обеспечивает?

Определение: Обеспечение соответствия продукта установленным требованиям и ожиданиям заинтересованных сторон.

АспектПример
КачествоСоответствие ISO 25010
РискиПредотвращение финансовых потерь

4. Модель качества программного обеспечения (ISO 9126)

Определение: Международный стандарт, определяющий 6 ключевых характеристик качества ПО.

  • Функциональность
  • Надежность
  • Эффективность

5. Стандарт ISO 9126. Описать.

Определение: Стандарт, предоставляющий framework для оценки качества программных продуктов.

Применение: Метрики для портируемости - 
время адаптации к новой платформе ≤ 8 часов

6. Программная ошибка

Определение: Несоответствие между фактическим результатом выполнения программы и ожидаемым поведением.

  • Классификация по severity:
    • Critical
    • Major

7. Классификация тестирования ПО

Определение: Систематизация методов тестирования по целям, уровням и используемым техникам.

ТипПример
ФункциональноеПроверка расчётов
НагрузочноеТест на 1000 пользователей

8. Компьютерная программа (ISO/IEC/IEEE 24765:2010)

Определение: Исполняемая последовательность инструкций, предназначенная для обработки данных.

Пример архитектуры:
Presentation Layer → Business Logic → Data Layer

9. Компьютерная программа (ISO/IEC 2382-1:1993)

Определение: Совокупность машинных инструкций, выполняющих определённую задачу.

Пример компиляции:
Исходный код → Объектный код → Исполняемый файл

10. Описать два подхода к компьютерной программе с точки зрения математики

Определение: Интерпретация программы как математического объекта с чётко определёнными свойствами.

  • Как функция: f(x) = x²
  • Как алгоритм: Последовательность шагов

11. Тест план.Выделите несколько пунктов, которые должны быть представлены в тест плане в соответствии с best practice.

Определение: Документ, описывающий цели, подходы, ресурсы и график тестовой деятельности.

  • Обязательные разделы:
    • Стратегия тестирования
    • Критерии завершения

13. Виды тестирования

Определение: Категоризация тестовых практик по целям и методам выполнения.

Примеры:
- Дымовое (Smoke)
- Регрессионное (Regression)

14. Реализация тестирования (этапы)

Определение: Процесс выполнения запланированных тестовых активностей в рамках жизненного цикла ПО.

  1. Планирование
  2. Проектирование тест-кейсов

15. Основные типы ошибок

Определение: Классификация дефектов по их природе и влиянию на систему.

Пример логической ошибки:
if (a > b) вместо if (a >= b)

16. Метод ручного тестирования

Определение: Метод тестирования, при котором тест-кейсы выполняются человеком без использования средств автоматизации.

  • Плюсы: Гибкость
  • Минусы: Трудоёмкость

17. Метод индукции

Определение: Логический вывод от частных случаев к общим закономерностям.

Пример: 
Тест 1: 2 → 4
Тест 2: 3 → 6
Вывод: Функция удваивает вход

18. Метод дедукции

Определение: Логический вывод от общих предпосылок к частным случаям.

Пример: 
Все API требуют аутентификации → 
Новый эндпоинт должен требовать токен

19. Обратное прослеживание

Определение: Метод отладки, при котором анализ начинается с точки сбоя до первопричины.

  1. Фиксация ошибки
  2. Анализ логов

20. Тестирование методом белого ящик

Определение: Метод тестирования с доступом к внутренней структуре и реализации системы.

Метрики:
- Покрытие кода ≥ 80%
- Сложность циклов ≤ 10

21. Согласно ISTQB (International Software Qualifications Board), тестирование методом белого ящика – это:

Определение: Техника проектирования тестов, основанная на анализе внутренней логики программы.

  • Критерии:
    • Покрытие условий
    • Покрытие путей

22. Тестирование мутаций – это

Определение: Метод оценки качества тестов путём внесения контролируемых изменений в исходный код.

Пример мутанта:
return a + b → return a * b

23. Тестирование по принципу «черного ящика»

Определение: Метод тестирования, при котором проверяется функциональность без знания внутренней структуры.

Пример:
Ввод невалидного email → Проверка сообщения об ошибке

24. Цели техники тестирования по принципу «черного ящика»

Определение: Проверка соответствия системы функциональным требованиям и спецификациям.

  • Задачи:
    • Валидация входных данных
    • Проверка граничных значений

25. Какие методы формирования тестовых наборов включает в себя стратегия «черного ящика»?

Определение: Стратегии генерации тестовых данных на основе спецификаций.

Примеры:
- Эквивалентное разделение
- Таблицы решений

26. Модульное тестирование – это:

Определение: Тестирование отдельных компонентов системы в изоляции.

Пример на Python:
assert sum([1,2,3]) == 6

27. Преимущество модульного тестирования

Определение: Преимущества изолированной проверки компонентов на ранних этапах разработки.

  • Раннее обнаружение ошибок
  • Упрощение отладки

28. Недостатки модульного тестирования

Определение: Ограничения методов тестирования отдельных модулей.

  • Не выявляет интеграционные ошибки
  • Требует поддержки тестов

29. Рекомендации по модульному тестированию

Определение: Best practices для эффективного модульного тестирования.

Пример:
Использование моков для внешних сервисов

29. Юзабилити-тестирование – это

Определение: Оценка удобства использования системы конечными пользователями.

  • Метрики:
    • Время выполнения задачи
    • Количество ошибок

30. Цель юзабилити-тестирования

Определение: Выявление проблем взаимодействия пользователя с интерфейсом.

Пример отчёта:
40% пользователей не нашли кнопку "Купить"

31. Основные моменты при проверке GUI

Определение: Тестирование графического интерфейса на соответствие требованиям.

  • Критерии:
    • Кросс-браузерность
    • Адаптивность

32. Ручное тестирование

Определение: Выполнение тестов человеком без использования скриптов.

Пример:
Проверка отображения на мобильных устройствах

33. Функциональное тестирование

Определение: Проверка соответствия системы функциональным требованиям.

Пример:
Проверка расчёта суммы заказа

34. Тестирование взаимодействия

Определение: Проверка корректности интеграции компонентов системы.

Пример:
Взаимодействие модуля оплаты с CRM

35. Тестирование безопасности

Определение: Проверка защиты системы от несанкционированного доступа.

  • Техники:
    • SQL-инъекции
    • XSS

36. Способов измерения покрытия структуры программы

Определение: Метрики оценки полноты выполнения элементов кода тестами.

Примеры:
- Statement Coverage
- Branch Coverage

35. Комплексное тестирование

Определение: Тестирование всей системы в условиях, приближенных к реальным.

Пример:
Проверка полного сценария покупки

36. Тестирование стрессов

Определение: Проверка работы системы при экстремальных нагрузках.

Пример:
5000 одновременных пользователей

37. Тестирование объема

Определение: Проверка обработки больших объёмов данных.

Пример:
Загрузка файла 50 ГБ

38. Тестирование конфигурации

Определение: Проверка работы системы в различных окружениях.

Пример:
Windows 11 / Ubuntu 22.04

39. Тестирование совместимости

Определение: Проверка работы системы с различными платформами и устройствами.

Пример:
Chrome, Firefox, Safari

40. Тестирование защиты

Определение: Проверка механизмов безопасности системы.

Пример:
Проверка HTTPS на всех страницах

41. Тестирование требований к памяти

Определение: Проверка корректного использования и освобождения памяти.

Инструменты:
Valgrind, Memory Profiler

42. Тестирование производительности

Определение: Проверка скорости работы и стабильности системы под нагрузкой.

Метрики:
Response time ≤ 2 сек

43. Тестирование настройки

Определение: Проверка процессов установки и конфигурации ПО.

Пример:
Установка через Docker

44. Тестирование надежности/готовности

Определение: Проверка стабильности работы системы в течение длительного времени.

Пример:
Нагрузка в течение 72 часов

45. Тестирование средств восстановления

Определение: Проверка способности системы восстанавливаться после сбоев.

Пример:
Восстановление из бэкапа

46. Тестирование удобства обслуживания.

Определение: Проверка удобства внесения изменений в систему.

Пример:
Время добавления новой функции

47. Тестирование публикаций

Определение: Проверка документации и справочных материалов.

Пример:
Руководство пользователя

48. Техника анализа граничных значений.

Определение: Техника тестирования на границах допустимых диапазонов.

Пример:
Минимальное/максимальное значение поля

49. Тест-требования

Определение: Формализованные условия, подлежащие проверке.

Пример:
TR-001: Система должна поддерживать 100 RPS

50. Тест-планы - это

Определение: Документ, описывающий стратегию, объём и график тестирования.

Пример раздела:
Критерии завершения: 95% покрытия требований
